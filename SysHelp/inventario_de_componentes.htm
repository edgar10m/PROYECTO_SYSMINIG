Inventario de Componentes
 
Este producto es el más importante dentro de Sys-Mining
©
ya que se considera como un diagnóstico de los componentes de la aplicación o aplicaciones a analizar y tiene las siguientes características:
Presenta el número de componentes a partir de los tres niveles básicos: Empresa, Oficina y Sistema, indicando para cada tipo (clase) el número de componentes.
Presentamos los siguientes inventarios:
 
INVENTARIO
DESCRIPCIÓN
Existente
Total de componentes físicos incorporados a la base de conocimiento (bc).
Faltante
El número de componentes que se “referencian” en algún otro pero del cual no tenemos su componente físico.
Sin uso
Componentes físicos no referenciados en ningún otro
Activos
Componentes físicos sí referenciados en algún otro componente
Identificados
El total de componentes referenciados sin considerar si contamos con el fuente o no
 
Principales Características:
 
  • • 
    • 
    El cálculo del inventario se actualiza en línea una sola vez cuando se realiza una Consulta de Inventario y además se ha integrado por lo menos un nuevo componente a la base de conocimiento.
  • • 
    • 
    A partir del tipo de inventario se puede imprimir o exportar a Excel el conjunto de componentes.
  • • 
    • 
    A cada componente del árbol le aplica un menú de opciones propias de su tipo que se presentan en una ventana independiente.

Inventario de Componentes tiene como objetivo, mostrar totales por cada tipo de componente asi como un total final de cada uno de ellos.
Los totales se muestran a nivel de sistema, oficina y por empresa.
La ventana de Inventario de Componentes contiene diferentes pestañas: Existentes, Faltantes, Sin Uso, Activos e Identificados y dependiendo de la que se elija, se darán los totales mencionados anteriormente.
La información consultada en el inventario puede enviarse a impresión ó exportarse a Excel.
 
Cuando se elige alguno de los componentes del inventario y dando doble clic aparece una ventana flotante o menú contextual a partir del cual se puede solicitar: análisis de impacto, diagrama de proceso, documentacion, lista de componentes, propiedades, referencias cruzadas,  ver fuente ó versiones.
