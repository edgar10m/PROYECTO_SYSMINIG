Recepción de Componentes
 
Este módulo nos permite realizar la carga manual (en línea) de los componentes que conforman una aplicación, es un auxiliar para la carga ya que existe una aplicación automática que permite realizar esta.
 
 
 
Mediante esta ventana, se podrá realizar la integración de componentes a la base de conocimiento de Sys-Mining.
 
Barra de menús Sistema
Barra de título y botón que contiene comandos para manipular la ventana, incluye los botones Minimizar, Restaurar y Cerrar ventana..
 
Arriba
Recepción
Directorio donde el usuario deposita los componentes para de allí tomarlos y posteriormente ser integrados en la base de conocimiento.
 
 
Arriba
Ubicación
Permite seleccionar una unidad de almacenamiento para realizar la recepción de componentes.
 
Arriba
Directorio
Indica el directorio donde se recibieron los componentes  para ser integrados en la base de conocimiento.
 
Arriba
Lista
Muestra la lista de componentes generados en el directorio espiecificado.
 
Arriba
Operación
En esta área se establecen:
- Características
- Destino y
- Consideraciones para los componentes
Una vez que se tienen los componentes proporcionados por el usuario, se debe determinar el destino de estos en la base de conocimiento, por lo que se deben especificar las características que se requiere aplicar a los componentes.
 
Arriba
Oficina
Se selecciona la oficina a la cual corresponde el componente que se va a integrar a la base de conocimiento.
 
Arriba
Sistema
Se selecciona el sistema al cual corresponde el componente que se va a integrar a la base de conocimiento.
 
Arriba
Componente
Tipo de Componente a elegir para la recepción en la base de conocimiento.
 
Arriba
Biblioteca
Se selecciona la biblioteca a la cual corresponde el componente que se va a integrar a la base de conocimiento.
En la caja de validación todas, cuando se encuentra activa, permite seleccionar todas las bibliotecas, así como también esta opción puede utilizarse durante una carga inicial de componentes en la base de conocimiento. En la caja de validación incluye subdirectorios, permite que la lista incluya más componentes.
 
Arriba
Máscara
La máscara es el patrón para seleccionar cierto rango de archivos por su nombre.
 
Arriba
Seleccionar todo
Selecciona todos los archivos listados en el área de recepción.
 
Arriba
Integra componentes
Activa la integración en la base de conocimiento de los componentes seleccionados desde el área de recepción.
 
Arriba
Parámetros adicionales
Se introducen directivas para una carga especial.
Se requiere conocer los comandos de la herramienta de minería.
Contiene una caja de validación la cual al estar activa permitirá introducir las directivas para una carga especial en el área de edición.
 
Arriba
Omite existentes
Si el componente ya existe en la base de conocimiento lo omite en la integración y continúa con el siguiente componente.
 
Arriba
Revisa Versiones
Verifica si ya existe el componente y lo compara con las versiones anteriores que han sido entregadas y pregunta si quiere ser agregado en caso de que exista una versión idéntica.
Esto aplica en situaciones cuando se entregan versiones antiguas por error.
 
Arriba
Analiza Fuente
Hay componentes que no necesitan ser analizados, por ejemplo, pantallas que no traen información sensible.
Analiza el fuente entonces será necesario para información sensible.
 
Arriba
Conserva Extensión
En caso de que se requiera que el componente se conserve con la extensión dentro de la base de conocimiento.
 
Arriba
Analiza Producción
Al seleccionar esta opción oculta el panel de recepción para dejar solo visibles los paneles de operación y resultados para tener una mejor viualización
 
Arriba
Verifica Clase
 
Arriba
Nombre_Version
Como su nombre lo dice, proporciona el nombre de la versión del componente.
 
Arriba
Integra Copys
 
Arriba
Reemplaza cadena
 
 
Arriba
Nombre Componente
Especifica como va a manejar el nombre del componente, en esta opción se puede especificar:
Nombre actual
Convirtiendo el nombre a mayúsculas
Convirtiendo el nombre a minúsculas.
 
Arriba
Log
Despliega en la segunda ventana del panel derecho, el log ó bitácora de operación.
Aparecen los errores en caso de que haya ó los totales de componentes que va dando de alta.
 
Arriba
Resultados
Despliega en cada uno de los paneles los diferentes resultados que proporciona la pantalla de Recepción de Componentes.
 
Arriba
Resumen
Contiene el Resumen de Caracteristicas  ó propiedades encontradas en los componentes.
También despliega algún error de la carga del componente.
 
Arriba
Contenido
Despliega el contenido del componente y también se utiliza para desplegar el Log de la operación.
El contenido del componente se despliega al dar clic en algún componente del área de recepción.
El contenido del Log se despliega al pulsar el botón Log del área de Operación.
 
Arriba
Versiones
Muestra las diferentes versiones que se han cargado de ese componente.
 
Arriba
Contenido
Despliega el Contenido de la biblioteca, una vez finalizada la carga, es decir, después de haber pulsado el botón de integración.
 
Arriba
Barra de desplazamiento
Permite moverse arriba o abajo en la ventana.
 
Arriba
Carga Utileria
Al activar este botón se abre la ventana de Utilerías Auxiliares a fin de agregar la utilería faltante.
 
Arriba
Todas las librerías
Abre una ventana de diálogo donde solicitar confirmar si se procesará todas las librerías listadas.
